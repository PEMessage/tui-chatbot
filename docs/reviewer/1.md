```python
if self._end_predicate and self._end_predicate(event):
```

```python
    if self._end_predicate and self._end_predicate(event):
        if self._result_extractor:
            result = self._result_extractor(event)
            self.end(result)
```

**问题**:
这是**最严重的设计问题**。`AgentEventType`, `AgentEvent`, `ChatResult` 在 core/events.py 和 agent/types.py 中都有定义，但结构不同：

```python
# core/events.py: AgentEvent 使用 Pydantic
class AgentEvent(BaseModel):
    type: AgentEventType
    ...

# agent/types.py: AgentEvent 使用 dataclass  
@dataclass
class AgentEvent:
    type: AgentEventType
    ...
```

这会导致:
1. 类型不一致错误
2. 导入困惑 (应该从哪里导入?)
3. 运行时序列化行为不同

**建议修复**:
1. 统一使用 core/events.py 的定义
2. agent/types.py 只保留消息类型定义 (AgentMessage, UserMessage, etc.)
3. 从 agent/types.py 导入事件类型时重定向到 core/events.py

```python
# agent/types.py 应该改为:
from ..core.events import AgentEventType, AgentEvent, ChatResult

# 只定义消息相关类型
class AgentMessage(ABC, BaseModel): ...
class UserMessage(AgentMessage): ...
...
```

## 问题清单（按严重程度排序）

### 🔴 高优先级

无

### 🟡 中优先级

1. **退出命令不正常工作**
   - 现象: `/quit` 显示 "Bye!" 但程序不退出
   - 影响: 用户需要按 Ctrl+C 或关闭终端才能退出
   - 建议: 修复 `ChatCommands.quit()` 方法，确保调用 `sys.exit(0)` 或正确关闭事件循环

2. **会话保存失败**
   - 现象: `/save` 显示 "No active session to save"
   - 影响: 用户无法保存当前对话
   - 建议: 检查 `SessionManager` 与 `REPL` 的集成，确保交互模式下正确创建和管理会话

3. **Assistant 回复偶尔不显示**
   - 现象: 第二条消息的回复只有推理过程，没有 [Assistant] 内容
   - 影响: 用户看不到 AI 的完整回复
   - 建议: 检查流式输出处理，确保 [Assistant] 标签后的内容被正确渲染

### 🟢 低优先级

1. **统计信息显示 0 tokens**（第一条消息）
   - 现象: 第一条消息显示 [0 TOKENS | 0.0s | TPS 0.0]
   - 可能原因: 模型未返回 token 使用信息，或解析问题
   - 严重程度: 低（不影响功能）

## 建议修复方案

### 修复 1: 退出命令
```python
# 在 ChatCommands.quit() 中
def quit(self) -> None:
    self.console.print("[dim]Bye![/dim]")
    # 添加以下代码确保程序退出
    import sys
    sys.exit(0)
```

### 修复 2: 会话保存
```python
# 在 REPL 类中确保 session 被正确初始化
class REPL:
    def __init__(...):
        self.session_manager = SessionManager(...)
        # 确保创建一个活跃会话
        self.current_session = self.session_manager.create_session(...)
        
    def handle_save(self):
        if self.current_session:
            self.session_manager.save_session(self.current_session)
```

### 修复 3: 输出渲染
```python
# 在流式输出处理中
if content.startswith("[Assistant]"):
    # 确保提取并显示 Assistant 内容
    assistant_content = content.replace("[Assistant]:", "").strip()
    if assistant_content:
        self.console.print(assistant_content)
```

### 2.1 工具调用测试

#### 2.1.1 Calculator 工具 (高优先级) ❌ 失败

**测试步骤:**
1. 启动聊天应用
2. 发送: "计算 123 * 456"
3. 观察是否触发工具调用
4. 验证结果是否正确显示 (56088)

**实际结果:**
```
>>> 计算 123 * 456
[Thinking | TTFT 0.0s | 0 tokens | TPS 0.0][Reasoning]: 
用户现在需要计算123乘以456，根据提供的工具，应该调用calculate工具...
[6 TOKENS | 100.0% REASONING | 0.0% CONTENT | 2.0s | TPS 3.0 | TTFT 0.80s]
>>>
```

**问题分析:**
- 模型输出了思考过程 (reasoning_content)
- 但**没有实际执行工具调用**
- 没有看到计算结果 56088
- 提示符直接返回，没有内容输出

**根本原因:**
`daemon.py` 直接使用 `provider.stream_chat()` 而不经过 `agent_loop`，导致工具调用事件被接收但**工具函数没有被执行**。

## 4. 根本原因分析

### 4.1 架构问题

```
当前流程 (有问题的):
main.py -> Shell -> Daemon.chat() -> provider.stream_chat()
                                      ↓
                                 工具事件被接收
                                      ↓
                                 工具函数未执行 ❌

正确流程应该是:
main.py -> Shell -> Daemon.chat() -> agent_loop()
                                      ↓
                              provider.stream_chat()
                                      ↓
                                 工具事件被接收
                                      ↓
                              _execute_tool_calls()
                                      ↓
                                 工具函数执行 ✅
```

### 4.2 代码问题定位

**文件:** `src/tui_chatbot/daemon.py`  
**问题:** `Daemon.chat()` 方法直接调用 `provider.stream_chat()`，没有使用 `agent_loop`

```python
# daemon.py (当前有问题的实现)
provider_stream = await provider.stream_chat(
    model=model,
    messages=self._messages,
    tools=tools,  # 工具定义传递了
    signal=signal,
)
# 只转发事件，但没有执行工具调用
async for event in provider_stream:
    stream.push(event)
```

**修复建议:** 使用 `agent_loop` 替换直接调用:

```python
from .agent import AgentLoopConfig, agent_loop
from .agent.types import UserMessage

# ...

config = AgentLoopConfig(
    tool_registry=self._tool_registry,
    max_iterations=5,
)

user_msg = UserMessage(content=text)
return await agent_loop(
    prompt=user_msg,
    history=self._messages,
    config=config,
    signal=signal,
)
```

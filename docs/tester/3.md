## 2. 失败的测试分析

### 2.1 类型 A：MockProvider 签名不匹配（20 个）

**根本原因**: `MockProvider.stream_chat()` 方法签名未更新以支持新增的 `temperature` 和 `max_tokens` 参数。

**影响测试**:

| 测试类 | 失败数量 | 错误信息 |
|--------|----------|----------|
| TestAgentLoopBasic | 4 | `TypeError: MockProvider.stream_chat() got an unexpected keyword argument 'temperature'` |
| TestAgentLoopWithTools | 3 | 同上 |
| TestAgentLoopMultiTools | 2 | 同上 |
| TestAgentLoopErrors | 3 | 同上 |
| TestAgentLoopStreaming | 2 | 同上 |
| TestAgentLoopConfig | 2 | 同上 |
| TestAgentLoopEdgeCases | 4 | 同上 |

**修复建议**:

修改 `tests/test_agent_loop.py` 中的 MockProvider 类：

```python
async def stream_chat(
    self,
    model: str,
    messages: List[dict],
    tools: Optional[List[dict]] = None,
    signal: Optional[Any] = None,
    temperature: Optional[float] = None,  # 新增
    max_tokens: Optional[int] = None,     # 新增
) -> EventStream[AgentEvent, ChatResult]:
    # 现有代码保持不变
```

根据设计方案，以下测试失败属于可接受的已知问题（优先级低）：

| 测试 | 失败原因 | 设计决策 |

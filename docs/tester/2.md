## 4. 失败测试分析

### 4.1 类型定义问题 (test_core_events.py) - 3 个失败

#### ❌ test_event_extra_fields_forbidden
```python
# 测试期望: AgentEvent 禁止额外字段
with pytest.raises(ValueError):
    AgentEvent(type=AgentEventType.AGENT_START, unknown_field="value")
# 实际: 没有抛出异常，AgentEvent 允许额外字段
```

**影响:** 低 - 不影响核心功能，只是数据验证严格性

#### ❌ test_result_is_immutable  
```python
# 测试期望: ChatResult 是不可变的 (frozen model)
with pytest.raises(Exception):
    result.content = "modified"
# 实际: ChatResult 是可变的
```

**影响:** 低 - 不影响核心功能，只是数据不变性保证

#### ❌ test_elapsed_zero_when_no_end
```python
# 测试期望: 当没有 end_time 时 elapsed 应该返回 0.0
stats = TokenStats(start_time=100.0)  # 设置 start_time
assert stats.elapsed == 0.0
# 实际: 返回了当前时间戳减去 start_time
```

**影响:** 低 - TokenStats 边界情况计算逻辑错误

### 4.2 测试配置问题 - 15 个失败

#### ❌ test_improvements.py (11 个测试)
**原因:** 所有 async 测试函数缺少 `@pytest.mark.asyncio` 装饰器

受影响的测试:
- test_event_stream_basic
- test_event_stream_result_first
- test_event_stream_dual_usage
- test_event_stream_empty
- test_abort_controller_basic
- test_abort_controller_wait
- test_abort_controller_timeout
- test_abort_controller_cancel_timeout
- test_event_stream_with_abort
- test_backward_compatibility
- test_event_stream_generic_types

#### ❌ test_integration.py (6 个测试)
**原因:** 同样缺少 `@pytest.mark.asyncio` 装饰器

受影响的测试:
- test_daemon_chat_returns_eventstream
- test_daemon_chat_early_abort
- test_eventstream_concurrent_usage
- test_eventstream_error_handling
- test_stream_with_no_result
- test_abort_during_iteration

**修复方案:**
在 pyproject.toml 添加全局 async 模式，或为每个测试函数添加装饰器:
```python
import pytest

@pytest.mark.asyncio
async def test_event_stream_basic():
    ...
```

### 4.3 向后兼容导入问题 (H2)

test_improvements.py 和 test_integration.py 从 `tui_chatbot.main` 导入:
```python
from tui_chatbot.main import EventStream, AbortController, Event, EventType
```

根据设计文档 H2，main.py 应该添加向后兼容导出，但目前实现情况未知。

## 1. 背景与目标

- 工具框架已完全实现 (`agent/tool.py`)

- 验证工具在真实对话中被正确触发和返回

- 测试工具调用失败时的错误处理
- 验证打断后系统状态的稳定性

## 3. 预期行为规范

### 3.1 工具调用流程

```
用户输入 → LLM 判断 → Tool Call → 执行工具 → 结果 → LLM → 回复
                    ↓              ↓
               [Tool: name]    result (绿色/灰色)
```

### 3.2 打断流程

```
生成中 → Ctrl-C → KeyboardInterrupt → CancelledError → 保存部分结果 → 恢复 prompt
         ↓
    [Interrupt] 显示
```

### 3.3 状态转换

| 状态 | 事件 | 下一状态 |
|------|------|----------|
| idle | 用户输入 | streaming |
| streaming | Ctrl-C | interrupted |
| streaming | 完成 | idle |
| interrupted | 新输入 | streaming |

## 9. 风险提示

| 风险 | 描述 | 缓解措施 |
|------|------|----------|
| LLM 工具识别率 | 简单模型可能不识别工具调用意图 | 使用支持工具的模型 (gpt-4-turbo, gpt-3.5-turbo-1106+) |
| 网络延迟 | 工具调用增加延迟 | 设置合理的超时 |
| 状态竞争 | 打断时可能有竞态条件 | 仔细测试 TC-12.6 |
